# Agentic AWS Auto-Remediation Pipeline (SOAR)

### Objective
A semi automated Security Orchestration, Automation, and Response (SOAR) pipelie designed to detect, audit, and autonomously remediate vulnerable AWS infrastructure using a custom Python AI agent with Human-in-the-Loop (HITL) authorization.

### Why I Built This
I built this pipeline to better understand and bridge the gap between traditional cloud security and agentic AI. My core objectives were to:
* **Learn AWS Infrastructure as Code:** Gain hands on experience with AWS as well as using Terraform to programmatically deploy and destroy cloud environments
* **Understand Cloud Misconfigurations:** Move beyond theory by physically building and exploiting common AWS vulnerabilities (exposed S3 buckets, open security groups)
* **Develop Agentic AI Tooling:** Write custom Python automation using Boto3 to prove that Security Orchestration, Automation, and Response (SOAR) can be handled autonomously rather than through manual console clicks (in a secure and controlled enviornment ofcourse)

## 📁 Repository Structure
* `main.tf`
  ** The Terraform configuration file that deploys the deliberately vulnerable AWS environment (VPC, Security Groups, S3 Buckets) 
* `prox_hitl.py`
  ** The Python agent. It ingests security telemetry, parses it for critical vulnerabilities, triggers the Telegram Human-in-the-Loop (HITL) prompt, and executes the Boto3 remediation commands
* `prowler_report.json`
  ** A sanitized snippet of the raw JSON output generated by the Prowler audit. I included this to demonstrate the exact data structure the AI agent is programmed to ingest and parse before making remediation decisions

## Architecture & Flow
1. **Infrastructure as Code (IaC):** Deployed a deliberately vulnerable AWS environment using Terraform (exposed S3 buckets, open SSH security groups).
2. **Cloud Security Audit:** Executed Prowler against the environment to benchmark against CIS foundations and generate vulnerability telemetry (JSON).
3. **Agentic Orchestration (Prox):** Built a custom Python agent utilizing Boto3 to ingest the Prowler telemetry, parse critical `FAIL` states, and map them to their corresponding AWS resources.
4. **Human-in-the-Loop (HITL):** The agent pauses execution and pushes an alert to a mobile device via the Telegram API detailing the CVEs and requesting authorization to deploy the fix.
5. **Autonomous Remediation:** Upon human approval (`Y`), the Agent dynamically executes the exact AWS CLI/Boto3 commands to isolate the S3 bucket and revoke the `0.0.0.0/0` SSH ingress rule, achieving zero touch cloud remediation.

## My Tech Stack

* **Security & Auditing:** Prowler, Pacu (Simulation)
* **Automation & Scripting:** Python 3, Boto3 (AWS SDK)
